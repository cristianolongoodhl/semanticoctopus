INTRODUCTION

The Semantic Octopus is a java application which allows one to publish a SPARQL endpoint providing a knowledge base obtained by aggregating multiple RDF knowledge bases from the web and performing subsequent reasoning. The Semantic Octopus is based on jena and pellet (ansell version).

COMPILE AND INSTALL

Semantic Octopus can be compiled from sources with maven (just run mvn install in the sources root). After compilation you will find a semanticoctopus-[vers]-jar-with-dependencies.jar file in the maven-generated target directory. The endpoint can be setup with the following command: 

java -jar semanticoctopus-version-jar-with-dependencies.jar endpointhost enpointport kburl1 [kburl2 [... kburln]]

where

  1. version is the semantic octopus version;
  2. endpointhost is the host name under which the aggregated ontology will be provided;
  3. enpointport is the endpoint port;
  4. kburl1, ..., kburln are the URLs of the ontologies to be aggregated.

For example, the following will start a sparql endpoint at port 9001 of localhost which provide the main opendatahacklab projects and events knowledge base. 

java -jar semanticoctopus-0.1.2-jar-with-dependencies.jar http://localhost 9000 http://opendatahacklab.org/site/odhl.owl
